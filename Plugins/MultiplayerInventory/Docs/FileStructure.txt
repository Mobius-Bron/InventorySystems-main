# MultiplayerInventory 插件 - 文件结构文档

## 整体概述
本插件是一个 Unreal Engine 5 多人库存系统插件,采用 MVVM 架构,
提供空间网格库存、装备系统、消耗品系统、物品拾取与丢弃等功能。
所有类的前缀均为 MIS_ (Multiplayer Inventory System),避免与现有项目命名冲突。

配套 DebugHelper 插件 (Plugins/DebugHelper) 提供运行时调试诊断,
在装备/拾取/丢弃等关键链路均有内置诊断输出。

## 核心架构层次

+--------------------------+
|       Widgets (UI)        |  ← 纯视图: 渲染 + 转发用户交互
+--------------------------+
|  MIS_InventoryComponent   |  ← ViewModel: 数据 + 命令 + 委托
|  MIS_EquipmentComponent   |
+--------------------------+
|  Fragments / Manifest     |  ← 数据模型: 物品属性配置
|  MIS_InventoryItem        |
+--------------------------+
|  MIS_FastArray            |  ← 网络层: 高效增量复制
+--------------------------+
|  Server RPC / Multicast   |  ← 网络通信: Client→Server→AllClients
+--------------------------+

## 关键设计决策

1. **组件附着策略**: InventoryComponent 和 EquipmentComponent 推荐放在 Character 上,
   EquipmentComponent 通过 Init(PC, InvComp, Mesh) 从外部传入所需引用,不依赖 Owner 类型。

2. **装备初始化**: 在 NotifyControllerChanged 中调用 Init (Controller 已就绪)。

3. **ProxyMesh 代理模式**: ProxyMesh 自带独立的 EquipmentComponent (bIsProxy=true),
   Init 时不执行属性修改效果,只负责 3D Actor 的视觉生成。

4. **网络通信**: 所有装备/卸下操作走 Server RPC → Multicast RPC 模式,
   确保服务端和所有客户端同步执行。
